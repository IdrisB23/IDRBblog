Stacks and Queues

Binary data cutting into line!

by Idris Babay on August 1st, 2020, 8min reads

Share now

I always look up the motivation behind every new concept I learn to know the struggles clever people have encountered in the past and how they came up with these ideas, and this would be my approach in all posts.

A stack follows the Last-In-First-Out (LIFO) principle. It can be thought of as a pile of pancakes; You'd have to (technically) eat those at the top first. It maintains a pointer S.top to its topmost element and the 2 main operations a stack supports are Stack-PUSH and Stack-POP (Yup, those are the official terms). PUSH(S, e) inserts the new element e at the top of the stack. POP(S) does exactly the inverse and removes the element at the top. Both operations update S.top accordingly.

Almost any modern piece of software supports action-undoing and -redoing. This can be achieved by maintaining 2 stacks: UNDO and REDO; each time an action is undone, we pop it from UNDO and push it onto REDO. With redoing, we conversely pop from REDO and push onto UNDO. We won't discuss here what data the action elements contains.

A queue follows the First-In-First-Out (FIFO) principle. Analogous to lines in supermarkets or anywhere basically, the person who has waited longest gets served first. Queues keep 2 pointers Q.head and Q.tail and, like stacks, they support 2 main modifying operations; ENQUEUE(Q, e) inserts e at the head of the queue and DEQUEUE(Q) removes the element at the tail.

Queues can be used to implement a variety of systems, where many operations are at hand and are required to be executed in the future in sequential order. It's used for example in Input/Output requests: when typing on the keyboard, the OS is adding pressed keys to a queue and handing them over to the software, in whose form you're typing.

Both of these data structures can be implemented using simpler ones that we have already studied, notably linked lists and arrays. With the former, we'd only have to properly call INSERT(L, e) and DELETE(L, e) to insert at top/head and delete from top/tail. Using arrays is, however, more subtle. For both structures we'll use an array A of size n.

A stack requires a pointer A.top that gives us the slot in the array that contains the top element. Each time we push onto A, we update A.top = A.top + 1 and add element at that slot. Popping means overriding the value with null (means empty or unset) and decreasing A.top by 1. If A.top = 0, the stack is empty. If we attempt to pop an empty a stack-underflow error occurs. Conversly, the stack is full when A.top = n and a push operation, then, results in a stack-overflow error, which is a very common error in programming environments. Try a similar approach to implement queues using arrays!

We will revisit stacks and queues and use them in future posts when we'll inspect graphs and graph traversal algorithms. Until then, stay healthy!
